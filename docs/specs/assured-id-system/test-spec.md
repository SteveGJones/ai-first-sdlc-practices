## Tests

### TEST-assured-id-system-001

**Title:** `parse_id`, `format_id`, and `is_positional` round-trip and error behaviour

**Description:** (a) Call `parse_id("REQ-my-feature-001")` and assert the returned `ParsedId` has `kind="REQ"`, `feature="my-feature"`, `number=1`, `program=None`, `sub_program=None`, `module=None`; assert `is_positional(parsed)` returns `False`; assert `format_id(parsed) == "REQ-my-feature-001"`. (b) Call `parse_id("P1.SP1.M2.DES-007")` and assert `kind="DES"`, `program="P1"`, `sub_program="SP1"`, `module="M2"`, `number=7`, `feature=None`; assert `is_positional(parsed)` returns `True`; assert `format_id(parsed) == "P1.SP1.M2.DES-007"`. (c) Assert `parse_id("INVALID")` raises `IdParseError`. (d) Assert `parse_id("REQ-007")` raises `IdParseError` (missing feature segment). (e) Assert `parse_id("P1.SP1.M2.REQ-feature-001")` raises `IdParseError` (positional form does not accept a feature segment). (f) Assert `parse_id("TEST-a-b-c-001")` succeeds with `feature="a-b-c"` (multi-segment feature slug). (g) Assert `parse_id("CODE-validated-gateway-042")` returns `kind="CODE"` and `number=42`.

**satisfies:** REQ-assured-id-system-001 via DES-assured-id-system-001

### TEST-assured-id-system-002

**Title:** `build_id_registry` idempotency, `render_id_registry` purity, and `remap_ids` path immutability

**Description:** (a) Using a temporary directory as `project_root`, write two minimal spec files under `docs/specs/` — one with a flat `### REQ-widget-001` heading and a `**satisfies:** REQ-widget-000` line, one with a positional `### P1.SP1.M1.REQ-001` heading and no satisfies line. Call `build_id_registry(project_root)` twice and assert the two returned lists are equal element-by-element (idempotency). Assert the list contains exactly two `IdRecord` values in the order matching sorted file enumeration. (b) Call `render_id_registry(records)` on the result and assert: the output starts with `<!-- Generated by kb-rebuild-indexes`, contains the `| ID | Kind | Source | Satisfies |` header row, and contains one row per record. Call it again on the same records and assert the output string is identical (purity). (c) Call `remap_ids(records, {"docs/specs/old/": "docs/specs/new/"})` on records whose `source` begins with `"docs/specs/old/"` and assert each returned record has its `source` updated to begin with `"docs/specs/new/"` while the `id` field is unchanged. Assert that records whose `source` does not match the prefix are returned with their `source` unmodified. Assert that no record's `satisfies` list is mutated between input and output.

**satisfies:** REQ-assured-id-system-002 via DES-assured-id-system-002
