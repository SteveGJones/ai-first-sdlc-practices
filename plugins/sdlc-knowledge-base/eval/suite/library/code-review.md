# Code Review

Code review is the practice of having a change examined by someone other than its author before it is merged. Every change should be reviewed before merge to catch defects and share knowledge across the team.

Effective reviews check for correctness, adequate tests, readability, and adherence to the team's coding standards. Reviewers should focus on the design and logic rather than purely cosmetic preferences.

Small pull requests are reviewed more thoroughly than large ones. Keeping a change under about four hundred lines helps reviewers stay focused and find more defects.
