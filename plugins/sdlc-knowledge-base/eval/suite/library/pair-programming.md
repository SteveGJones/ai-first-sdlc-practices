# Pair Programming

Pair programming is a practice where two developers work together at one workstation. One developer, the driver, writes the code while the other, the navigator, reviews each line as it is written.

The two developers switch roles frequently so that both stay engaged. Pair programming spreads knowledge across the team and can reduce defects because review happens continuously rather than after the fact.

Pair programming is most valuable for complex or unfamiliar work. For routine tasks the cost of two people on one keyboard may outweigh the benefit.
