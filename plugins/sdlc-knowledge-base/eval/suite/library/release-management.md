# Release Management

Release management is the process of planning, scheduling, and controlling the movement of a software release through its stages to production. Its goal is to deliver value safely and predictably.

Semantic versioning assigns a three-part version number of the form major dot minor dot patch. The major version increases for incompatible changes, the minor version for backward-compatible features, and the patch version for backward-compatible bug fixes.

A release should be accompanied by a changelog that records the notable changes since the previous release. Teams often freeze the codebase before a release to stabilise it, allowing only critical fixes during the freeze.
