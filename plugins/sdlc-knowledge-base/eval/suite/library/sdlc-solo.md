# Solo SDLC Method

The solo method is the lightest of the four SDLC methods and is designed for a single developer working alone. It keeps ceremony to a minimum so that one person can move quickly.

The solo method still requires a feature proposal and a retrospective for each piece of work. It uses feature branches and forbids committing directly to the main branch.

The solo method is appropriate for prototypes, personal projects, and small tools. When a second contributor joins the project, teams should migrate from the solo method to the single-team method.
